# シリアル通信プロトコル (Serial Communication Protocol)

このソフトウェアでは、外部ソフトウェア（PC側のコントローラー制御アプリなど）とMCU（Raspberry Pi Pico）の間でUART（シリアル通信）を用いて情報を送受信します。この通信におけるデータのフォーマットと制約について解説します。

## 概要

### 通信設定
- ボーレート: 9600 bps
- ポート: UART0 (GP0/TX, GP1/RX)

すべてのコマンドは、末尾に改行文字 `\n` (`\r\n`) を付与することでターミネートされ、MCU側での解釈が実行されます。

---

## 1. 標準コントローラー入力 (`PC_CALL`)

外部ソフトウェアからNintendo Switchに対して動的なボタン入力やジョイスティック操作を行わせるための汎用フォーマットです。以下のように、各入力値をスペース区切りの16進数文字列で送信します。

**基本フォーマット:**
`<Buttons(Hex)> <Hat(Hex)> [LX(Hex) LY(Hex)] [RX(Hex) RY(Hex)]\r\n`
*(※ `[ ]` で囲まれた部分は、`<Buttons(Hex)>` で指定するスティック更新フラグの状態によって**省略可能**です)*

**各項目の詳細:**

- **`<Buttons(Hex)>` (ボタン入力フラグとスティックフラグ)**
  - ボタンの押下状態は1つの16ビット値の中で論理和（OR演算）されて表現されます（例：Aボタン + Bボタン）。
  - **フォーマット制約**: このフィールドは必ず **`0x` を含む6文字（例: `0x001B`）**の形式になるようにゼロ拡張して送信する必要があります。
  - **ビットレイアウト**:
    - `0x0001` (ビット0): **右**スティック(`RX`/`RY`)の更新フラグ。これが `0` の場合、後続の `<RX(Hex)> <RY(Hex)>` 指定は**省略可能（不要）**です。
    - `0x0002` (ビット1): **左**スティック(`LX`/`LY`)の更新フラグ。これが `0` の場合、後続の `<LX(Hex)> <LY(Hex)>` 指定は**省略可能（不要）**です。
    - `0x0004` (ビット2): Yボタン
    - `0x0008` (ビット3): Bボタン
    - `0x0010` (ビット4): Aボタン
    - `0x0020` (ビット5): Xボタン
    - `0x0040` (ビット6): Lボタン
    - `0x0080` (ビット7): Rボタン
    - `0x0100` (ビット8): ZLボタン
    - `0x0200` (ビット9): ZRボタン
    - `0x0400` (ビット10): MINUSボタン
    - `0x0800` (ビット11): PLUSボタン
    - `0x1000` (ビット12): LCLICKボタン
    - `0x2000` (ビット13): RCLICKボタン
    - `0x4000` (ビット14): HOMEボタン
    - `0x8000` (ビット15): CAPTUREボタン
    - *(例：Aボタン（`0x0010`）を押下しつつ、両スティックの更新フラグ（`0x0003`）を指定する場合、送信値は `0x0013` となります。)*
  
- **`<Hat(Hex)>` (十字キー)**
  - 十字キー（HAT）の入力方向は0〜8までの値を取る8ビットの16進数で表現されます。
  - 値は以下の通り定義されています:
    - `0`: 上 (TOP)
    - `1`: 右上 (TOP_RIGHT)
    - `2`: 右 (RIGHT)
    - `3`: 右下 (BOTTOM_RIGHT)
    - `4`: 下 (BOTTOM)
    - `5`: 左下 (BOTTOM_LEFT)
    - `6`: 左 (LEFT)
    - `7`: 左上 (TOP_LEFT)
    - `8`: 未入力/中心 (CENTER)

- **`<LX(Hex)>` / `<LY(Hex)>` / `<RX(Hex)>` / `<RY(Hex)>` (ジョイスティック)**
  - これらのパラメータには、0〜255 (`00`~`FF`) の8ビット16進数がセットされます。
  - 未入力のニュートラル状態は `128` (`80` / `0x80`) となります。
  - **省略ルール**: `<Buttons(Hex)>` のビット0/ビット1でフラグを立てていない場合、該当するスティックの座標パラメータは送信を**省略**できます。
    - **両方省略**: `0x00... 8\r\n` (ボタンと十字キーのみ送信)
    - **左のみ送信**: `0x00... 8 80 80\r\n`
    - **両方送信**: `0x00... 8 80 80 80 80\r\n`

**例1. ボタンのみ（スティック省略）**
`0x0010 8\r\n`
*(Aボタン（`0x0010`）のみ入力。スティック更新フラグ（最下位2ビット）が `0` のため、ジョイスティックの座標パラメータ群は全て省略しています)*

**例2. ボタンと左スティックのみ**
`0x0012 8 FF 80\r\n`
*(Aボタン入力 ＋ 左スティック更新フラグ（`0x0002`）で合計 `0x0012`。左スティックをX=255/限界右, Y=128/中央 に設定。右スティックの更新フラグが0のため、RX/RY成分の送信は省略)*

**例3. 十字キーと右スティックのみ**
`0x0001 2 80 FF\r\n`
*(ボタン入力はなしだが右スティック更新フラグ（`0x0001`）を指定。十字キーを右（`2`）に指定。右スティックをX=128/中央, Y=255/限界下 に設定。左スティック成分の送信は省略)*

**例4. フル送信（全要素の更新）**
`0x001B 8 00 00 FF FF\r\n`
*(AボタンとBボタン（`0x0010`+`0x0008`=`0x0018`） ＋ 両スティック更新フラグ（`0x0003`）で合計 `0x001B`。左スティックを左上に限界、右スティックを右下に限界まで倒すフルパラメータ指定)*

## 2. キーボード入力モード

このプロジェクトでは、Nintendo Switch等のキーボード入力が可能な環境において、文字入力機能を補助するためのキーボードエミュレーションモードも備えられています。

### 文字列送信 (`PC_CALL_STRING`)
- **フォーマット**: `"<string>"\r\n` または `"<string>"`
- **概要**: 先頭がダブルクォーテーション `"` である場合、受信した文字列をそのままASCIIテキストとして解釈しキーストロークにマッピングして出力します。

### キー修飾子・単独キー指令 (`PC_CALL_KEYBOARD` 系)
特殊なキー（Enter、Shiftなど）を入力するためのコマンド群です。

- **`Key <num>\r\n`** : 特定のキーコード（通常はHIDキーコード）を受け取り一瞬押下して離す動作 (`specialkey_write`)
- **`Press <num>\r\n`** : 特定のキーを押しっぱなしにする動作 (`specialkey_press`)
- **`Release <num>\r\n`** : 特定のキーの押下を解除する動作 (`specialkey_release`)

## 3. MCU制御・マクロ関数起動用のコマンド

MCU内にあらかじめプログラムされた操作パターンを実行するためのトリガー文字列です（例：マクロの自動実行）。これらは特定の文字列に完全一致した場合のみ起動します。

- `"mash_a\r\n"` や `"pickupberry\r\n"` など、プレーンテキスト形式で送信されます。このコマンドの動作や詳細については、`docs/mcu_commands.md` を参照してください。
